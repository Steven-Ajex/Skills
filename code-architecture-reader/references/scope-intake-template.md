# 范围确认模板（Scope Intake Template）

当用户只说“帮我看看这个项目架构”而没有给出范围、深度或目标时，使用此模板快速确认。

## 1. 分析目标（Goal）

优先确认用户更关心哪一类：

1. 快速上手（建立整体认知）
2. 重构前评估（找风险与改进点）
3. 代码评审（架构层面的优缺点）
4. 某条链路追踪（定位流程与边界）
5. 某类机制理解（缓存/消息/鉴权/插件等）

## 2. 范围（Scope）

至少确认一个：

1. 全项目
2. 某个目录/模块
3. 某条功能链路
4. 某次变更（PR/commit）影响范围

## 3. 输出档位（Depth）

1. 快读版：10-20 分钟可读摘要
2. 标准版：可用于评审/重构讨论
3. 深度版：完整架构理解报告（含证据索引、残余风险）

## 4. 约束（Constraints）

- 是否只看静态代码（不运行项目）？
- 是否有时间限制？
- 是否需要特别关注某类问题（性能、并发、扩展性、测试性）？
- 是否需要中文输出 + 英文术语注释（默认是）？

## 5. 最小默认策略（用户不回复时）

采用默认值：

1. 范围：整体架构 + 1 条核心链路
2. 档位：标准版
3. 重点：边界、耦合、可测试性、扩展性
4. 明确声明抽样范围与不确定项
