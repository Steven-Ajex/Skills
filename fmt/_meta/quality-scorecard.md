# FMT Skills 质量评分卡（Quality Scorecard）

用于评估和持续优化 `fmt/` 目录下 skills 的质量。可用于新增 skill 自检，也可用于旧 skill 重构优先级排序。

## 1. 评分方式

每项 `0-2` 分，总分 `10` 分：

1. `0` 分：缺失或明显不足
2. `1` 分：有定义，但不够可执行/可验证
3. `2` 分：清晰、可执行、可验证

## 2. 评分维度（5 项）

1. 触发准确性（Trigger Accuracy）
2. 边界纯度（Boundary Purity）
3. 输入输出契约（IO Contract Clarity）
4. 证据可追溯性（Evidence Traceability）
5. 失败与降级策略（Failure/Fallback）

## 3. 分数解释

1. `9-10`：可稳定复用，优先保持边界不扩张
2. `7-8`：可用，但建议补模板/门禁/交接工件
3. `5-6`：容易误用或跳步，建议优先重构
4. `<5`：高风险技能，建议拆分或重写

## 4. FMT 原子技能额外加分检查（非强制）

1. 是否明确区分代码证据与日志证据
2. 是否明确区分变体（`vtol/mc/fw`）适用范围
3. 是否给出下游技能可消费的工件名称与字段
4. 是否包含至少一个专业误判防护规则

## 5. 评估建议流程

1. 先按评分卡打分（不要先改文件）
2. 识别最低分维度
3. 只做“最小有效改造”提高该维度
4. 回归检查是否破坏边界纯度
